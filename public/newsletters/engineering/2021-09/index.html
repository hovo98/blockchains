Tendermint BFT & Core

Led by Interchain GmbH and Informal Systems

Since the last report:

  • we have released Tendermint 0.35, the latest major version which includes prioritized mempool, clean up of the p2p layer, state sync improvements and the option of using PostgreSQL as an indexer. More can be found here.
  • In recent months we have also pathed out the Tendermint Roadmap leading to the 1.0 version. More can be found here.

In regards to core engineering, we’ve made several improvements to increase the reliability and effectiveness of the Tendermint end-to-end tests. We have cleaned up swaths of the node package, removed the legacy p2p stack, made various incremental adjustments to the RPC, Websocket and the Events system, and finally introduced basic rollback functionality.

Looking onwards, our next priorities are:

  • The completion of ABCI++, a redesign of the application boundary which introduces four new methods: PrepareProposal, ProcessProposal, ExtendVote and FinalizeBlock. This essentially adds more hooks into the consensus phases and is intended to unlock a plethora of new use cases including threshold decryption, concurrent processing, and immediate execution. The implementation work here will be accompanied with updated specs and documentation as well as an internal audit for what is perhaps the most highly anticipated feature for the 0.36 release.

  • The implementation of proposer based timestamps as outlined in ADR071 which replaces the existing BFTTime with a more reliable and resistant form of timekeeping. The crux of the change involves the proposer proposing a time as part of a block and the validators within the network agreeing on the time so long as it falls with in a certain proximity to the validators own local clock.

  • The outlining of the Soft Upgrades protocol which aims to ensure backwards compatible chain upgrades so that historical chain data is maintained across upgrades. This also involves setting up more formal processes to upgrading and allowing for greater automation.

              Cosmos Cash

              Led by Tendermint

              Since last report:

              1. We have finalized the design for the chain of trust that will allow e-money token issuers to issue collateralize stable coins within a regulatory framework. The design and implementation utilizes the self-sovereign identity (SSI) approach to guarantee regulatory compliance (ADR-004, ADR-005, ADR-006, ADR-007)

              2. Released v2.0.0 that improve DID standard compatibility and on-chain public verifiable credentials for the chain of trust.

              3. Released v2.1.0 that improves compatibility with Aries agent.

              4. Launched the documentation portal

              5. Launched a universal resolver driver for Cosmos Cash DID resolution

              Next priorities:

              1. Define a fungibility mechanism for multiple stablecoins backed by the same currency

              2. Implement a complete integration of cosmos-cash payment chain and Hyperledger Aries agents for identity and credentials
